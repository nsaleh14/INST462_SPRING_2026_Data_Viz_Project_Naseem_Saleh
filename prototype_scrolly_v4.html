Low contrast: 85.3% in 2019, 79.1% in 2025. Missing alt text: 68% in 2019, 55.5% in 2025. Unlabeled forms: 52.8% to 48.2%. Empty links: 58.1% to 45.4%. Empty buttons: 25% to 29.6% — only type increasing. Missing language: 33.1% to 15.8%.

These failures aren't distributed equally. Some industries have nearly twice as many as others.

So what about the developers who are trying to fix it?

There's a tool called ARIA designed specifically to make websites more accessible. 4 in 5 sites now use it. The results are not what you'd expect.

The Fix That Isn't Working

The fix that isn't working · 2025

Even when developers try to fix it, they often make it worse

ARIA is a set of tools developers use to make websites more accessible. 4 in 5 pages now use it. But pages using ARIA average more than twice the errors of pages that don't.

A second symptom

Most pages also have broken heading structure — the skeleton screen readers rely on

Screen readers use headings like a table of contents to navigate a page. When the structure is broken, blind users lose their map. Click each stat to see what that looks like.

ARIA and heading structure are fixable — but only if developers understand the rules. The data suggests that many are adding tools without the knowledge to use them correctly. The web is trying to become more accessible. It's just not succeeding.
